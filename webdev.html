  • cloud_queueASP.NET MVC Web Application

    Languages Used: C#, HTML, CSS , Javascript

    Frameworks: ASP.NET MVC, ASP.NET Razor, Materialize

    The website was developed in Visual Studio 2017, using the ASP.NET MVC Framework. ASP.NET Razor was used for the back end, whereas, the Materialize Framework was used for the front end.

    The models for this project included heroes, and their respective information and a repository which these heroes were stored in. The views for this project included the Index page as well as the Details page for a selected hero. The controllers for this project included the home controller as well as a controller for 404 errors.

    When a hero is chosen on the home page , the hero's name is passed to an action method which pulls the respective hero from the repository. The hero is then passed to the view which dynamically generates the details page for the hero.

    The web application also features filters which allow specific hero types to be highlighted on the home page. The solution took advantage of ids within HTML and Javascript to update the home page.

    Lastly, the web application was deployed and published through Microsoft Azure. Github was used for version control.

  • webPortfolio Website

    Languages Used: Javascript, HTML, CSS

    Team Size: 1

    Role: Front-End Developer

    The website was developed using the Materialize Framework which includes Jquery. I used the Materialize framework due to its expanded features and customization over Bootstrap. Both frameworks utilize a grid which makes creating layouts for websites much easier.

    The banner and nameplate were designed by me in Adobe Photoshop. I tried to keep the sizes of all included images on the page below 400kb to keep page load times low. Originally I was going to use screenshots of each individual project but it would have taken a large amount of space and time to do. Instead I used resizable icons to give my pages a professional look.

    The navbar is pinned to the top after a certain distance down the page. I altered the Javascript files included in the Materialize framework to do this. The navbar is also moved to a side navigation if the screen size meets a certain threshold.

    The Night Mode switch included in both the main navigation and side-navigation uses Javascript to overwrite classes on the website. By simply changing the class of the body element of the web page and adding the appropriate CSS the color scheme of the page is changed. In order to keep the color consistent across the different pages I used local storage over cookies.

    Lastly the Javascript files were compressed in order to improve load times of the pages.

  • web_assetWebsite UCR

    Languages Used: JavaScript, HTML, CSS

    Team Size: 3

    Role: Front-End Developer, Video Editor

    The website developed within a month for international students coming to UCR.

    The website was simple in design but was altered slightly to make it appear different from other pages on the engineering portion of the UCR site. The banner was swapped out for a new design using Javascript.

    The banner was designed in Adobe Photoshop by me. The team was told not to alter the slogan “We Engineer Excellence” as well as the college name. The fonts used were not available for free so I had to isolate portions of the original banner with the specified text before making changes.

    Part of the way through development of the page I was handed multiple large videos to add to the site. The videos were too large to add to the page and contained audio issues. I reduced a large amount of background noise using Adobe Premiere. I then reduced the size of the videos using compression software. The videos were reduced from 4GB to 20MB each.

    HTML5 video was used for displaying videos over embedding videos from Youtube onto the website. The choice was made due to Youtube not being available in certain regions. HTML5 video is supported by most modern browsers so it was an easier solution.

    Improvements to the website that could be made include responsiveness of the website. For example on the blog page the tables do not resize with the window or center its content so it is still viewable. However, the given specifications for the project restricted any extensive modifications to the layout of the website.

    The website was one of my early front-end development projects.

  • searchTwitter Search Engine

    Languages Used: Java, HTML, CSS

    Team Size: 2

    Role: Front-End Developer

    The search engine was developed using the Netbeans IDE and Lucene software.

    Tweets were gathered and stored using Twitter Streaming API . The gathered tweets were then parsed and stored using Lucene.

    Search queries were received through text fields in HTML. The query was then parsed and search results were generated through Lucene and Java. The resulting tweets were then embedded into a dynamically generated HTML file using Java Servlets. If a tweet included a geolocation, the location would be displayed on a map.

    The website was not mobile friendly, but could be fixed through proper usage of classes and rules within the CSS for the website.

    The search engine was hosted locally and deployed within the Netbeans IDE.
