  • vpn_keyPassword Cracker

    Languages Used: Python

    Team Size: 2

    Role: Programmer

    On Linux systems encrypted passwords are stored within an etc/shadow file. For the project the password length was known to be 1-6 characters long.

    To find a matching password all possible password combinations had to be generated then encrypted using the same method as the etc/shadow file. The Linux system uses the md5 encryption method which is contained within a crypt library for Python.

    To increase the throughput of generated passwords I first opted to use threading in order to test multiple passwords at once. However, Python has a Global Interpreter Lock (GIL) which limits execution so that only one thread is running at a time. To circumvent this limitation I used the multiprocessing library in Python.

    The way I generated the passwords was extremely slow. The generated passwords go through the same sequence of characters repeatedly, so I could have used a lookup table to create passwords quicker. Preferably I would have stored all passwords up to 3 characters then concatenated them together once I reached passwords of 6 characters length. The process would reduce the number of concatenations immensely in comparison to the ridiculous amount of nested loops I used.

  • peopleLinkedIn Desktop Application

    Languages Used: Java , PostgreSQL

    Team Size: 2

    Role: User Interface Designer

    Professional Network application which handles:
    • New User Registration
    • User Login/Logout
    • Change Password
    • Search People
    • Sends Connection Request
    • Accepts or Rejects Connection Request Connection Request
    • View Friends and then go to a friends profile.
    • Send a message to anyone on network.
    • View Messages and then option to delete messages

    I developed the UI for the application in Java using a combination of the AWT and Swing libraries. To show the UI the frames had to be redrawn after actions such as button presses were performed. For certain portions of the application queries were performed within the database and displayed within the UI.

    Given the opportunity to work on this project again I would have opted for a java library which allowed for easier formatting of windows as some portions of the application varied in size. Either the Box Layout or Grid Layouts would have sufficed.
